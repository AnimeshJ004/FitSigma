Date: July 14, 2026
Issue: Changes visible on localhost but not on Railway deployment

================================================================================
ROOT CAUSE IDENTIFIED
================================================================================

Laravel caches several types of files that can cause old versions to persist:

1. Compiled Views (storage/framework/views/*.php)
2. Configuration Cache (bootstrap/cache/config.php)
3. Route Cache (bootstrap/cache/routes.php)
4. Application Cache (storage/framework/cache/*)

When deploying to Railway, these caches were not being properly cleared,
causing your application to serve old, cached content even though the source
code was updated.

================================================================================
TECHNICAL DETAILS
================================================================================

Old start.sh behavior:
  php artisan config:clear
  php artisan cache:clear
  php artisan route:clear
  php artisan view:clear

Issue: These commands only clear the Laravel cache registry, but compiled
files may still exist on disk if the cache clearing fails or is incomplete.

New start.sh behavior:
  1. Run all artisan clear commands
  2. Manually delete all files in cache directories
  3. Recreate cache directories
  4. Ensure proper permissions

Result: Guaranteed fresh compilation on every deployment.
